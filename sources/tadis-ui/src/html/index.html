Info:
  • A box represents i) a microservice or ii) a microservice and its outgoing exchange if the microservice and its outgoing exchange have the same name.
  • Box colors have the following meaning:
    • regular microservice
    • microservice with the shared property set
    • indicates that the source code location of the microservice is unknown. thus no source analysis can be carried out.
    • indicates that an exchange with the same name as the microservice exists. this exchange is merged into the microservice node. the exchange can be either incoming or outgoing.
  • A gray cylinder represents a RabbitMQ message exchange.
  • A black link represents an asynchronous information flow from source to target. If both source and target are connected to a microservice then a message exchange exists in between them. This message exchange is not shown.
  • A red link represents a synchronous information flow (note: the actual synchronous call takes place in the opposite direction).
